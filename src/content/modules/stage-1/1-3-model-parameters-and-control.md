## 为什么学这个

同一个 prompt，模型有时稳定、有时发散，往往不是“今天状态不好”，而是参数在背后决定了采样方式。不会调参数，你就很难解释为什么一个任务应该要严谨输出，另一个任务却需要更多创造性。

这一课的重点不是死记参数定义，而是建立控制面板意识：当输出太随机、太长、停不下来或格式不稳时，你知道该从哪里下手，而不是盲目重写 prompt。

## 核心知识

Temperature 控制采样时的随机性。值越低，模型越倾向选择概率最高的 token，输出更稳定、更像“标准答案”；值越高，模型探索空间更大，更容易出现多样表达，也更容易偏题。对于分类、抽取、改写成固定格式这类任务，通常先从低 temperature 开始；对于文案、创意发散、多个方案生成，可以适当提高。

Top-p 是另一种控制采样范围的方法。它不是直接放大或缩小随机性，而是先保留累积概率达到某个阈值的一小部分候选 token，再从里面采样。实践上，temperature 和 top-p 都能影响“发散程度”，但通常不建议同时大幅调整两个参数，否则很难判断问题到底来自哪里。先固定一个，再观察另一个，更容易建立直觉。

Max tokens 用于限制输出长度，它既关系到费用，也关系到体验。太小会让回答被截断，太大则可能导致模型啰嗦甚至跑偏。Stop sequences 则像人为设置的“终止标记”，当模型生成到指定字符串时立刻停止，适合多段输出、脚本协议或你想让模型只填写某一段内容的场景。真正有效的调参方式不是凭感觉，而是拿同一个 prompt，在不同参数配置下对比输出的稳定性、格式一致性和成本。

## 关键对比 / 易错点

- Temperature 低不等于“更正确”，它只意味着更接近高概率答案。如果训练数据里高概率模式本身就错，仍然会稳定地错。
- Top-p 和 temperature 都会影响随机性，但机制不同。新手最容易犯的错是同时乱调，最后无法定位原因。
- Max tokens 是输出上限，不是“期望长度”。不给更明确的格式或字数约束，模型仍可能在上限内过度展开。
- Stop sequence 必须和你预期的输出协议匹配。设置错了，可能导致答案过早截断或根本不生效。
- 参数不是 prompt 的替代品。任务定义本身模糊时，调再多参数也很难稳定。

## 学习检查点

- 如果你要让模型做事实抽取，为什么通常不会把 temperature 设得很高？
- Top-p 和 temperature 的核心区别是什么？为什么调参时最好一次只重点观察一个？
- 当输出经常被截断时，你应该先检查哪些可能原因？
- 什么场景下 stop sequences 比单纯缩短 max tokens 更合适？

## 实践建议

- 为同一个 prompt 设计三组参数组合，例如“稳定版”“平衡版”“发散版”，比较输出差异并做简短记录。
- 选一个需要固定格式的任务，分别尝试只改 prompt、只改参数、两者都改，观察哪个环节真正解决了问题。
