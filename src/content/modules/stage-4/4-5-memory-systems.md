## 为什么学这个

Agent 一旦开始执行多步任务，就不再只是“看当前这句话回答当前这句话”。它需要记住刚才查到了什么、用户偏好是什么、哪些中间假设已经被证伪、哪些线索值得后面继续跟进。如果没有记忆系统，Agent 每一步都像失忆一样重新开始；但如果什么都记、永不清理，系统又会被历史噪声拖垮。

因此，记忆设计不是单纯“把聊天记录塞回上下文”这么简单，而是一个状态管理问题。你要学会区分哪些信息只在当前任务有用，哪些值得长期保存，哪些只是中间草稿用完就该丢。只有把短期记忆、长期记忆和工作记忆分开理解，后续的 Agent 才既不会健忘，也不会啰嗦和混乱。

## 核心知识

短期记忆最常见的是对话历史管理。最原始的 Buffer Memory 会把全部历史顺序保留，优点是实现简单，缺点是上下文迅速膨胀；Window Memory 只保留最近若干轮，适合短对话但容易丢失远端关键信息；Token Buffer 则按 token 预算截断，更贴近真实模型成本限制。它们解决的是“当前这轮推理需要带多少近期上下文”，而不是长期个性化记忆。

长期记忆关注的是跨会话、跨任务保留有价值的信息。常见做法有两类。第一类是向量化记忆，把历史片段嵌入后按语义检索，适合找回“你之前提过的偏好”或“上次研究结论中的相似主题”；第二类是结构化存储，比如把用户资料、项目状态、显式偏好、待办事项写进数据库或键值表，更适合需要精确字段和稳定更新的场景。向量记忆擅长模糊找回，结构化记忆擅长精确读取和修改，二者经常组合使用。

工作记忆常被称为 Scratchpad，它不是给长期保留的，而是给当前任务推理过程临时放草稿、子问题、待验证假设和中间观察的地方。你可以把它理解成 Agent 的“草稿纸”。它的价值在于避免模型把所有中间推理都混进最终对话历史，也让多步任务中的阶段性结论有一个单独容器。工作记忆写得太少，复杂任务容易丢步骤；写得太杂，又会让后续轮次被无关推理污染。

写入与遗忘策略是真正决定记忆质量的关键。不是每句话都值得写进长期存储。一个常见原则是: 只有在信息稳定、可复用、对未来决策有价值时才写入，例如长期偏好、常用项目背景、明确确认过的事实。对于短期历史，可以用摘要压缩代替原文堆积；对于长期记忆，要设计更新和过期机制，避免旧偏好、过时事实一直污染系统。忘记并不是缺陷，而是控制噪声和成本的必要能力。

## 关键对比 / 易错点

- 聊天历史不等于完整记忆系统。短期上下文、长期记忆和工作记忆解决的是不同问题。
- Buffer 留得最全，但不代表最好；历史越长，成本和噪声通常一起上升。
- 向量记忆适合“语义找回”，结构化记忆适合“精确状态”；混用边界不清会让读取逻辑混乱。
- Scratchpad 是给当前任务的草稿纸，不是所有中间推理都应该沉淀成长期记忆。
- 写入策略比存储技术更重要。没有写入门槛，记忆系统很快会从资产变成负担。

## 学习检查点

- Buffer、Window、Token Buffer 三种短期记忆策略分别在解决什么问题？
- 长期记忆为什么不能只靠“把所有历史做向量化”来解决？
- 工作记忆和长期记忆的最大差别是什么？哪些信息更适合只放在 Scratchpad？
- 为什么说“遗忘策略”是记忆系统的一部分，而不是补丁？

## 实践建议

- 为你计划中的 Agent 先写一张记忆分类表，把“当前轮对话”“用户稳定偏好”“任务中间草稿”分别放到不同槽位，避免一开始就把状态混在一起。
- 给长期记忆定义明确写入规则，例如“只有用户明确确认的偏好才保存”“超过两周未命中的临时事实自动过期”，先从简单规则做起。
